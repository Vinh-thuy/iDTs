Descriptions de Poste - Simulation Insight

Data Scientist – Simulation & Prédiction d’incidents

Objectif

Anticiper les incidents et effets de bord grâce à des simulations hybrides basées sur graphes, historiques et règles métiers.

Missions

  • Développer des modèles prédictifs d’incidents
  • Simuler des scénarios “what-if”
  • Cartographier les dépendances dans un graphe
  • Exploiter les historiques incidents / changements

Compétences clés

  • Python, scikit-learn, PyTorch
  • Séries temporelles, événements
  • Neo4j / Cypher
  • Pathway ou moteur de simulation
  • Connaissances IT (réseaux, serveurs, cloud)

Data Scientist – Modélisation comportementale & Effets de cascade

Objectif

Modéliser les comportements dynamiques de l’infrastructure IT, comprendre les effets de bord et chaînages d’incidents, prévenir les dégradations progressives.

Missions

  • Développer des modèles de propagation
  • Identifier des signaux faibles
  • Créer des graphes d’interaction
  • Simuler des scénarios de charge

Compétences clés

  • Séries temporelles : Prophet, Kats, tsfresh
  • Propagation sur graphes
  • API Dynatrace, Elastic
  • Infrastructure IT distribuée

Data Scientist – Recommandation & Mémoire contextuelle (LLM)

Objectif

Structurer une mémoire intelligente pour les agents LLM, enrichir les requêtes utilisateurs, et personnaliser les recommandations selon le contexte.

Missions

  • Mettre en place un RAG performant
  • Modéliser les préférences utilisateur
  • Créer un agent mémoire adaptatif
  • Gérer l’interaction avec les agents

Compétences clés

  • LangChain, LlamaIndex, LanceDB
  • Embeddings OpenAI, Mistral
  • NLP : intentions, entités
  • Graphes de connaissance
  • Architectures RAG

Lead Simulation Architect – Chef de file Simulation Insight

Objectif

Piloter la vision technique et fonctionnelle de la couche Simulation Insight, organiser les cas d’usage, fédérer les data scientists et assurer la cohérence globale.

Missions

  • Définir l’architecture des modules
  • Coordonner les équipes
  • Valider les structurations de base
  • Piloter les ateliers et cas d’usage
  • Traduire les besoins métiers

Compétences clés

  • ITSM, CMDB, systèmes IT
  • Simulation de systèmes complexes
  • LLM + graph + symbolic AI
  • Leadership produit

Data Engineer – Pipeline & Orchestration

Objectif

Fiabiliser la collecte et l’orchestration des données opérationnelles pour la simulation.

Missions

  • Développer des pipelines résilients
  • Intégration Dynatrace, ServiceNow, Elasticsearch
  • Orchestration avec Prefect / Airflow
  • Gestion qualité, logs, alertes

Compétences clés

  • Prefect, Airflow, Pathway
  • RabbitMQ, APIs
  • Pipeline monitoring
  • JSON, Parquet, streaming

Data Engineer – Graph & Base de connaissance IT

Objectif

Construire et maintenir la base graphe de l’infrastructure et des dépendances.

Missions

  • Structurer la base Neo4j
  • Modéliser dépendances et incidents
  • Exposer APIs Cypher, REST
  • Calculs de propagation, centralité

Compétences clés

  • Neo4j / Cypher, TigerGraph
  • Ontologies IT, CMDB
  • Bases vectorielles (Milvus, LanceDB)
  • Relations et causalité
